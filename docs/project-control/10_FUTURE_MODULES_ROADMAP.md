# Future Modules Roadmap

آخرین به‌روزرسانی: 2026-06-28

## هدف

این سند نقشه آینده کل سیستم‌عامل هوشمند مستر جم را ثبت می‌کند. این نقشه فقط برای تصمیم‌سازی مرکز کنترل است و به معنی شروع ساخت ماژول، ادغام کد، migration، تغییر route یا تغییر localStorage نیست.

## تصمیم‌های قطعی فعلی

- هیچ ماژول جدیدی قبل از کاهش بدهی معماری `WorkforcePages.tsx` شروع نشود.
- پروژه پول / `audit-app` فعلاً Subproject بماند و کد آن merge نشود.
- پروژه کالا / `mahak-web-version` فعلاً Subproject بماند و کد آن merge نشود.
- فقط مدل‌ها، schemaها، ایده‌ها و الگوهای مفید بعداً با تأیید مرکز کنترل استخراج شوند.
- `Core Product Model` باید قبل از اتصال کالا، تولید، انبار و محک طراحی شود.
- `Core Financial Event Model` باید قبل از اتصال پول، رسیدها و رویدادهای مالی طراحی شود.
- اپ موبایل بعد از مشخص شدن مدل رسیدها و رویداد مالی شروع شود.
- تولید و انبار بعد از مدل کالا شروع شوند.
- اتصال محک بعد از تثبیت مدل کالا و خروجی‌ها بررسی شود.

## ترتیب پیشنهادی آینده

1. کاهش بدهی معماری WF و کوچک‌سازی ادامه‌دار `WorkforcePages.tsx`.
2. طراحی `Core Product Model`.
3. طراحی `Core Financial Event Model`.
4. طراحی `Core Inventory Model`.
5. طراحی `Receipt Capture Flow`.
6. طراحی `Mahak Integration Boundary`.
7. طراحی `Central Cockpit Dashboard Map`.
8. بررسی Mobile Capture App پس از روشن شدن مدل رسید و رویداد مالی.

## قواعد توقف

- اگر working tree تمیز نیست، هیچ نقشه یا فاز جدیدی شروع نشود.
- اگر یک کار نیاز به route/storage/model جدید دارد، باید فاز مستقل و تأیید مرکز کنترل داشته باشد.
- اگر کاری باعث merge مستقیم زیرپروژه‌ها می‌شود، متوقف شود و فقط به سطح extract idea/model/schema برگردد.
