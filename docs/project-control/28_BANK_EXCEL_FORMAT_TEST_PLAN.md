# Bank Excel Format Test Plan

آخرین به‌روزرسانی: 2026-06-29

## هدف

این سند طرح تست امنیتی برای فایل Excel گردش حساب بانکی آینده را ثبت می‌کند. هدف این است که تغییر قالب، خطای ستون، مبلغ نامعتبر، تاریخ خراب یا فایل تکراری قبل از ورود داده مالی تشخیص داده شود. این فقط سند است و هیچ parser یا import واقعی ایجاد نمی‌کند.

## ستون‌هایی که باید شناسایی شوند

| ستون | اهمیت |
|---|---|
| تاریخ | ضروری برای ترتیب و match زمانی |
| ساعت | تکمیلی برای تشخیص تراکنش‌های نزدیک |
| سند | شناسه بانکی احتمالی |
| شماره قبض | شناسه پرداخت یا قبض |
| شماره چک | اتصال به چک یا تعهد مالی |
| توضیحات | ورودی اصلی rule matcher |
| برداشت | مبلغ خروجی |
| واریز | مبلغ ورودی |
| مانده | کنترل sanity و ترتیب تراکنش‌ها |
| یادداشت | توضیح تکمیلی احتمالی |
| شماره پیگیری | معیار قوی برای duplicate و exact match |
| مبدا | حساب یا طرف مبدا |
| مقصد | حساب یا طرف مقصد |

## تست‌های قالب و parsing

| تست | انتظار |
|---|---|
| تست تغییر قالب اکسل | تغییر نام ستون، ترتیب ستون یا ساختار فایل باید گزارش شود و import خودکار متوقف شود. |
| تست نبودن ستون مهم | نبود تاریخ، توضیحات، برداشت/واریز یا شماره پیگیری باید warning یا error بسازد. |
| تست جابه‌جایی ستون‌ها | normalizer باید با mapping امن تشخیص دهد یا import را به review ببرد. |
| تست عددی نبودن مبلغ | مقدار غیرعددی در برداشت یا واریز باید parse error ایجاد کند. |
| تست تاریخ نامعتبر | تاریخ غیرقابل تبدیل باید error شود. |
| تست توضیحات خالی | تراکنش بدون توضیح نباید وارد matcher خودکار شود. |
| تست برداشت/واریز همزمان | وجود همزمان برداشت و واریز در یک ردیف باید conflict باشد. |
| تست duplicate transaction | شماره پیگیری، مبلغ، تاریخ و حساب مشابه باید duplicate candidate شود. |
| تست فایل تکراری | checksum یا metadata فایل تکراری باید import را متوقف کند یا به review بفرستد. |
| تست مانده ناسازگار | اختلاف مانده با ترتیب برداشت/واریز باید warning بسازد. |
| تست encoding یا متن ناخوانا | متن توضیحات خراب یا ناخوانا باید parse warning ایجاد کند. |

## خروجی‌های مورد انتظار

| خروجی | توضیح |
|---|---|
| `BankExcelFormatReport` | گزارش قالب فایل، ستون‌های شناخته‌شده، ستون‌های گمشده و نسخه قالب. |
| `BankExcelParseWarning` | هشدار غیرمسدودکننده مثل توضیح خالی یا ستون تکمیلی ناقص. |
| `BankExcelParseError` | خطای مسدودکننده مثل تاریخ نامعتبر یا مبلغ غیرعددی. |
| `BankExcelImportDecision` | تصمیم نهایی import: allowed، blocked، needs review یا format approval required. |

## معیارهای توقف import

- ستون تاریخ یا مبلغ قابل تشخیص نباشد.
- هر دو ستون برداشت و واریز در یک ردیف مقدار داشته باشند.
- مبلغ یا تاریخ قابل parse نباشد.
- فایل با checksum یا metadata قبلاً import شده باشد.
- قالب فایل با نسخه تاییدشده تفاوت جدی داشته باشد.
- تعداد خطاها از آستانه مصوب آینده بیشتر شود.

## تصمیم‌های ممنوع فعلی

- ساخت reader واقعی Excel.
- import مستقیم فایل به database اصلی.
- auto confirm بر اساس فایل با قالب ناشناخته.
- تغییر database، migration، auth، route یا localStorage.
