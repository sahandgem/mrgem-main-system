# Bank Rule Management and Installment Audit

آخرین به‌روزرسانی: 2026-06-29

## هدف

این سند مشخص می‌کند قانون‌های تشخیص تراکنش بانکی آینده چگونه توسط مدیر تعریف، نسخه‌بندی، غیرفعال و audit شوند و تایید خودکار قسط چه ردپایی باید داشته باشد. این مرحله فقط مستند است و هیچ rule engine اجرایی ساخته نمی‌شود.

## Bank Rule Management

قانون بانکی یک تعریف مدیریتی است که به سیستم کمک می‌کند از توضیحات، مبلغ، جهت تراکنش و اطلاعات حساب، رویداد مالی یا قسط احتمالی را تشخیص دهد.

## قانون‌هایی که مدیر می‌تواند تعریف کند

| قانون | توضیح |
|---|---|
| عبارت داخل توضیحات | کلمه یا عبارت دقیق/تقریبی در شرح بانکی. |
| مبلغ دقیق یا بازه مبلغ | مبلغ ثابت قسط یا بازه قابل قبول برای پرداخت/دریافت. |
| برداشت/واریز | جهت تراکنش برای جلوگیری از اشتباه دریافت و پرداخت. |
| حساب مبدا | حساب یا طرف پرداخت‌کننده احتمالی. |
| حساب مقصد | حساب یا طرف دریافت‌کننده احتمالی. |
| طرف حساب احتمالی | مشتری، فروشنده، کارمند یا طرف قرارداد. |
| شماره قرارداد | اتصال تراکنش به قرارداد یا توافق مشخص. |
| شماره قسط | تشخیص installment مشخص. |
| بازه تاریخ | محدوده زمانی قابل قبول برای match. |
| اقدام مورد انتظار | ثبت رویداد، تایید قسط، اتصال به رویداد موجود، review یا فقط هشدار. |

## نسخه‌بندی rule

هر rule باید این اطلاعات را در آینده داشته باشد:

- شناسه rule.
- نسخه rule.
- شخص ایجادکننده یا تغییر‌دهنده.
- تاریخ ایجاد و تغییر.
- دلیل تغییر.
- وضعیت فعال/غیرفعال.
- سطح ریسک.
- مجوز auto confirm یا manual only.
- آخرین نتیجه audit.

## Installment Auto Confirmation Audit

هر تایید قسط باید audit trail داشته باشد. برای auto confirm باید ثبت شود:

- کدام rule باعث match شد.
- rule در چه نسخه‌ای بود.
- مبلغ، تاریخ، جهت تراکنش و شماره پیگیری چه مقدار داشت.
- چرا سیستم آن را exact یا strong match دانست.
- آیا duplicate check انجام شد یا نه.
- آیا receipt/bank conflict وجود داشت یا نه.
- چه کسی rule را تعریف یا تایید کرده بود.
- زمان تایید خودکار.
- امکان بازبینی و لغو تصمیم در آینده.

## تغییر، حذف یا غیرفعال کردن rule

هر تغییر rule باید لاگ شود:

| اقدام | الزام audit |
|---|---|
| ایجاد rule | دلیل، محدوده، اقدام مورد انتظار و سطح ریسک ثبت شود. |
| تغییر rule | مقدار قبلی و جدید، دلیل و شخص تغییر‌دهنده ثبت شود. |
| غیرفعال کردن rule | دلیل توقف و اثر روی auto confirmهای آینده ثبت شود. |
| حذف rule | حذف واقعی توصیه نمی‌شود؛ بهتر است archived یا inactive شود. |
| تغییر مجوز auto confirm | باید نیازمند تایید مدیر باشد. |

## چه چیزهایی نباید auto confirm شوند؟

- پرداخت حساس.
- اختلاف مبلغ.
- اختلاف تاریخ زیاد.
- duplicate candidate.
- تراکنش بدون شرح کافی.
- تراکنش بدون rule مصوب.
- تراکنش با conflict.
- فایل Excel با قالب تاییدنشده.
- rule تازه‌ساخته‌شده بدون سابقه یا تایید کافی.
- match فقط بر اساس شباهت متن توضیحات.
