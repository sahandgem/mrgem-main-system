# Document Status and Approval Model

آخرین به‌روزرسانی: 2026-06-29

## هدف

این سند مدل وضعیت سند و مدل تایید مدیر را برای هسته مستر جم طراحی می‌کند. هدف این است که هر سند از ایجاد تا بررسی، اصلاح، تایید، نهایی‌سازی، لغو یا آرشیو، مسیر روشن و قابل audit داشته باشد.

این سند فقط طراحی معماری است و هیچ کد اجرایی، route، UI، database، migration، auth، localStorage یا schema محک ایجاد نمی‌کند.

## Document Status Model چیست؟

`Document Status Model` چرخه وضعیت سند را تعریف می‌کند. این مدل مشخص می‌کند سند در هر لحظه قابل ویرایش، قابل بررسی، قابل تایید، نهایی، لغوشده یا آرشیوشده است.

## Approval Status Model چیست؟

`Approval Status Model` مشخص می‌کند آیا سند نیاز به تایید مدیر دارد، چه کسی باید آن را تایید کند و نتیجه تایید چه بوده است. این مدل برای سندهای حساس، پرداخت‌های بزرگ، اصلاحیه‌ها، rollback و تغییر موجودی حیاتی است.

## وضعیت‌های پیشنهادی سند

| وضعیت | معنی |
|---|---|
| `draft` | سند در حال تهیه است و هنوز وارد جریان رسمی نشده است. |
| `submitted` | سند برای بررسی ارسال شده است. |
| `under_review` | reviewer یا manager در حال بررسی سند است. |
| `correction_requested` | اصلاح داده، سند، item یا mapping درخواست شده است. |
| `approved` | سند تایید شده اما ممکن است هنوز finalized نشده باشد. |
| `rejected` | سند رد شده و وارد جریان اصلی نمی‌شود. |
| `finalized` | سند نهایی شده و فقط از مسیر correction/rollback قابل تغییر است. |
| `cancelled` | سند لغو شده اما trace آن باقی می‌ماند. |
| `archived` | سند برای نگه‌داری تاریخی یا reference آرشیو شده است. |

## وضعیت‌های تایید

| وضعیت | معنی |
|---|---|
| `not_required` | سند نیاز به تایید ندارد. |
| `required` | سند نیاز به تایید دارد اما هنوز ارسال نشده است. |
| `pending_manager` | در انتظار تصمیم مدیر است. |
| `approved_by_manager` | مدیر تایید کرده است. |
| `rejected_by_manager` | مدیر رد کرده است. |
| `escalated` | سند برای تصمیم بالاتر یا بررسی عمیق‌تر ارجاع شده است. |

## کدام سندها approval می‌خواهند؟

- پرداخت حساس.
- دریافت/پرداخت بزرگ.
- اصلاحیه مالی.
- سند تولید حساس.
- تغییر موجودی حساس.
- merge کالا.
- rollback.
- حذف یا لغو سند.
- سند دارای conflict یا low confidence.
- سند import شده از داده خارجی با ریسک بالا.

## قوانین

- سند حساس بدون approval نهایی نشود.
- conflict یا low confidence نباید auto approve شود.
- AI فقط پیشنهاد بدهد، تصمیم حساس با مدیر باشد.
- هر تغییر status باید audit trail داشته باشد.
- سند finalized فقط از مسیر correction یا rollback قابل تغییر است.
- cancelled و archived نباید باعث حذف trace شوند.

## ارتباط با BaseDocument

فیلدهای `status` و `approvalStatus` در BaseDocument مفهومی باید از این مدل تبعیت کنند. این طراحی هنوز schema اجرایی نیست و قبل از هر database/migration باید فاز مستقل داشته باشد.
