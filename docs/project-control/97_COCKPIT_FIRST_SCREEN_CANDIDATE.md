# Cockpit First Screen Candidate

آخرین به‌روزرسانی: 2026-06-29

## وضعیت

این سند اولین screen candidate پیشنهادی برای prototype آینده cockpit را ثبت می‌کند. این انتخاب فقط planning است و ساخت prototype یا UI واقعی را تایید نمی‌کند.

## Cockpit First Screen Candidate چیست؟

First Screen Candidate صفحه‌ای است که اگر مرکز کنترل بعداً اجازه prototype ایزوله بدهد، نقطه شروع کم‌ریسک‌تر و قابل بررسی‌تر برای طراحی cockpit خواهد بود.

## پیشنهاد first screen

**Central Cockpit Overview Screen**

## دلیل انتخاب

- نقطه مرکزی تصمیم مدیر است.
- همه workstreamها را به‌صورت خلاصه نشان می‌دهد.
- می‌تواند با mock data ساخته شود.
- قبل از drill-downهای پیچیده قابل بررسی است.
- برای فهم ۱۰ ثانیه‌ای وضعیت مناسب است.
- به Design Lab اجازه می‌دهد card layout، risk/confidence language و manager decision shortcuts را بدون اتصال واقعی تست کند.

## کارت‌های مجاز در first screen concept

- Financial Pressure
- Product Import Warning
- Manager Review Queue
- Inventory Shortage
- Production Risk
- Workforce Risk
- AI Suggestion
- Crisis Signal

## کارت‌هایی که فعلاً فقط placeholder هستند

- Bank Import Status
- Installment Confirmation
- Mobile Receipt Queue
- Cash-in/Cash-out

## داده‌های مجاز

- synthetic/mock cockpit signals
- mock review counts
- mock risk levels
- mock confidence levels
- mock AI suggestions

## داده‌های ممنوع

- داده واقعی مالی
- داده واقعی کالا
- داده واقعی بانک
- داده واقعی محک
- داده واقعی کارمند
- داده واقعی مشتری یا فروش

## خروجی آینده در صورت approval

- Cockpit Overview Mock Screen
- Cockpit Card Layout Study
- Cockpit Risk/Confidence Visual Test
- Cockpit Manager Decision Mock

## قانون

این candidate فقط مسیر پیشنهادی آینده است. هیچ route، component، UI، storage، data binding یا prototype واقعی در این فاز ساخته نمی‌شود.
